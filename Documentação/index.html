Autenticação (POST)

Endpoint: /api/oauth/token


# Curl de exemplo:
curl --location --request POST 'http://127.0.0.1/api/oauth/token' \
--header 'Content-Type: application/x-www-form-urlencoded' \
--data-urlencode 'login={login}' \
--data-urlencode 'senha={senha}'

Lista - Adicionar (POST)

Endpoint: /api/lista/adicionar


# Curl de exemplo:
curl --location --request POST 'http://127.0.0.1/api/lista/adicionar' \
--header 'Authorization: Bearer {access_token}' \
--form 'titulo="Teste do título"' \
--form 'descricao="Somente um teste de envio da descrição da lista"'
                


Resultado exemplo :

{
    "dados": "Dados foram inseridos com sucesso.",
    "codigoLista": "20"
}

HEADER PARÂMETROS

Campo Tipo Descrição
Authorization Bearer {access_token} Token gerado pela autenticação do usuário

QUERY PARÂMETROS

Campo Tipo Descrição
titulo * String(150) Título da lista
descricao * Text Descrição da lista
* Campos obrigatórios

Lista - Atualizar (PUT, PATCH)

Endpoint: /api/lista/atualizar/{codigo_lista}


# Curl de exemplo:
curl --location --request PUT 'http://127.0.0.1/api//lista/atualizar/{codigo_lista}' \
--header 'Authorization: Bearer {access_token}' \
--header 'Content-Type: application/x-www-form-urlencoded' \
--data-urlencode 'titulo=Teste' \
--data-urlencode 'descricao=Mais um teste'

curl --location --request PATCH 'http://127.0.0.1/api//lista/atualizar/{codigo_lista}' \
--header 'Authorization: Bearer {access_token}' \
--header 'Content-Type: application/x-www-form-urlencoded' \
--data-urlencode 'titulo=Teste' \
--data-urlencode 'descricao=Mais um teste'
                


Resultado exemplo :

{
    "dados": "Dados atualizados com sucesso."
}

HEADER PARÂMETROS

Campo Tipo Descrição
Authorization Bearer {access_token} Token gerado pela autenticação do usuário
Content-Type application/x-www-form-urlencoded

QUERY PARÂMETROS

Campo Tipo Descrição
codigo_lista* INT Código da lista no sistema
titulo * String(150) Título da lista
descricao * Text Descrição da lista
* Campos obrigatórios pelo menos a passagem de um parâmetro

Erros

Código de Erro HTTP URL Mensagem
400 O recurso solicitado não está disponível
401 Access_token é inválido, verifique se cabeçalho novamente.
401 Header com problema faltando Content-Type: application/x-www-form-urlencoded.
401 Access_token inválido. Deve ser gerado um novo para o consumo da API.
404 O recurso solicitado não está disponível
400 /lista/consultar/{codigo_lista}, /lista/excluir/{codigo_lista}, /lista/atualizar/{codigo_lista} A variavel informada deve ser do tipo inteiro.
400 /oauth/token Login ou senha informados não são válidos!
400 /oauth/token Login e senha são obrigatórios
406 /lista/adicionar Houve algum erro ao inserir os dados.
400 /lista/adicionar Campos obrigatórios não informados
